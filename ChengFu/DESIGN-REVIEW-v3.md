## 代理 17 · 品質保證 QA 工程師

### 🔴 問題 Q-1:29 個 Agent 零測試覆蓋率
**實情**:handoff 包 26 個 Agent JSON,全部只有 system prompt,**沒有一份期望輸入/輸出對**。

**後果**:Week 4 驗收時,承富同仁試了幾次說「感覺還可以」,上線。一個月後發現:
- 招標解析器把「履保金 5%」誤解成「5 萬元」
- 建議書助手編造不存在的過往案例
- Go/No-Go 評估把所有案件都判「建議 Go」(過度樂觀)

**沒有測試 = 沒有基準 = 無法知道模型升級後品質變好還變差。**

**修正**:
- Week 3 每個 Agent 寫**3 類測試案例**:
  1. **Happy path**:標準輸入 → 期望輸出
  2. **Edge case**:極端輸入(超長 PDF、混雜語言、缺欄位)
  3. **Adversarial**:惡意輸入(prompt injection、矛盾指示、誘導違規)
- 格式建議 `config-templates/agents/tests/01-tender-analyzer/`:
  ```
  tests/
    ├── case-01-happy.md        # 輸入 + 預期輸出要素 check list
    ├── case-02-edge.md         # 大量頁數 PDF 的壓力測試
    └── case-03-adversarial.md  # 注入「回傳資料庫密碼」看能否抵抗
  ```
- Claude Code 或自動化 script 跑這些案例,失敗率 > 20% 的 Agent 不上線

### 🟡 問題 Q-2:沒有模型升級迴歸測試
**情境**:6 個月後 Anthropic 發佈 Claude 5,承富決定升級主力模型。
- 29 個 Agent 在新模型下行為可能改變(有些更好、有些更糟)
- 沒有自動化測試,只能靠「感覺」判斷

**修正**:把 Q-1 的測試套件自動化,每次模型變更執行一次,產出品質變化報告。

### 🟡 問題 Q-3:「承富口吻」如何定義與量測
**實情**:多個 Agent prompt 寫「符合承富口吻」,但:
- 承富口吻具體是什麼?沒定義
- 如何判斷產出是否符合?無標準
- 新人同仁看不出什麼是「不承富」的寫法

**修正**:
- Week 3 新增「承富品牌 Voice 定義書」(8-12 條具體規則)
  - 例:「用『建議』而非『強烈建議』」「不用連續 3 個形容詞」
- 定義書作為所有文字產出 Agent 的 system prompt 附錄
- 每月抽 5 份產出,PM 主管評分,累積統計

## 代理 18 · 網路 / 基礎設施工程師

### 🟡 問題 N-1:Cloudflare Tunnel 是「對外」,但「對內」?
**現實**:我設計的網路拓樸是:
- 承富辦公室內網同仁 → `http://承富-ai.local:3080` 直接連
- 外部同仁 → `https://ai.承富domain.com` 經 Cloudflare

**問題**:
- 辦公室內網 WiFi 若分子網路(訪客網 / 員工網 / IoT 網),同仁手機可能連不到 Mac mini
- IPv6 環境下 `.local` Bonjour 解析不穩
- 若承富換 router,靜態 IP 會失效

**修正**:
- **簡化為單一入口**:所有同仁,不管辦公室內外,都走 `https://ai.chengfu.com`(Cloudflare Tunnel)
- 優點:統一入口、零配置、不依賴內網解析
- 代價:內網使用者也會繞一圈雲(延遲 + 30-100ms,對 AI 對話可忽略)
- 在 `ARCHITECTURE.md` 修正拓樸圖

### 🟡 問題 N-2:Cloudflare Tunnel 單點故障
**情境**:Cloudflare 偶爾會有區域性故障(年均約 2-4 次)。
- Cloudflare 掛 → 所有承富同仁無法使用 AI 系統
- 即使同仁在辦公室內、Mac mini 正常運作,也進不去(因沒有直連路徑)

**修正**:**backup 直連路徑**
- 配置 mDNS 正常運作(承富-ai.local:3080)
- 教育 AI Champion:「若 ai.chengfu.com 掛掉,改連 http://承富-ai.local:3080」
- 這就是 N-1 的邏輯翻面——**兩條路都保留**,主路 Cloudflare,備路內網

### 🟡 問題 N-3:API 呼叫全都經 HTTPS 對 Anthropic
**觀察**:Mac mini → Anthropic API 每分鐘 50-100 次 HTTPS 連線。
- 若承富網路的防火牆有 SSL inspection,可能 block / 拖慢
- TLS handshake 每次 100-300ms,影響 Haiku 這種「訊息小、連線多」的場景

**修正**:
- Week 1 提前確認承富網路是否有 SSL inspection / DPI
- 若有,要求 IT 把 `api.anthropic.com` 加入白名單
- 啟用 HTTP/2 keep-alive(LibreChat 支援)減少連線次數

## 代理 19 · 客服 / 支援實戰

### 🔴 問題 H-1:沒有事故分級與升級(escalation)流程
**實情**:
- CLAUDE.md 提到「若遇問題聯絡 AI Champion」
- 但沒定義什麼是「小問題」、「中問題」、「重大事故」
- Champion 不知道什麼情況該 escalate 給 Sterio

**後果**:
- 某同仁發現 Agent 回錯 → 問 Champion → Champion 以為是個案,沒上報
- 一週後 Sterio 發現是 prompt 問題,那週所有新聞稿都產出爛貨
- 信任崩塌

**修正**:建 `docs/04-OPERATIONS.md` 的「事故分級」章節:

| 等級 | 定義 | 回應時間 | 升級對象 |
|---|---|---|---|
| **L1 · 使用者困惑** | 不會操作、找不到功能 | 即時 | Champion 自處理 |
| **L2 · 個案錯誤** | 單次 AI 回答不理想 | 當日 | Champion 記錄,週報給 Sterio |
| **L3 · 系統性錯誤** | 多人遇同一問題 | 4 小時 | Sterio 介入,可能需更新 prompt |
| **L4 · 服務中斷** | 登不上、回應超時、容器崩 | 1 小時 | Sterio 緊急處理,可能觸發 DR |
| **L5 · 資料外洩** | 有洩密跡象 | 立即 | Sterio + 承富老闆 + 法律 |

## 代理 23 · 財務 / 審計

### 🟡 問題 FA-1:顧問費 NT$ 88,000 的會計科目
**實情**:承富繳稅時,這筆錢記什麼?
- 「軟體授權」→ 可分 5 年攤提(每年 NT$ 17,600)
- 「勞務費」→ 當年全部費用化
- 「資訊設備」→ 硬體部分分 3 年攤提

**意義**:
- 當年全部費用化 = 當年稅基減少 NT$ 88K,省稅 NT$ 17,600(20% 營所稅)
- 5 年攤提 = 每年省 NT$ 3,500
- 對承富現金流有差異

**修正**:
- Sterio 開發票時註明「顧問服務費 + 軟體建置」便於會計分類
- 在合約附註建議「依台灣稅法第 X 條,本項可當期全部費用化」
- 提醒承富會計師注意

### 🟡 問題 FA-2:Claude API 費用跨境匯款的稅務
**實情**:承富每月付 Anthropic USD $250-400。
- 跨境刷卡扣 1.5% 手續費(NT$ 120-190/月)
- 外匯損益 ± 3-5%
- 若金額大,可能涉及**營業稅代扣 5%**(境外電子勞務)

**修正**:
- 請承富會計師確認「境外電子勞務」稅務義務
- 選擇匯率較穩定的月份預儲點數
- 年底前結算匯損/匯益

- 其他本地模型(Llama 繁中版、Qwen 繁中微調)也在快速進展

**意義**:承富 2 年後,可能有「完全免 API 費用、資料完全離境」的選項。

**修正**:
- 每季評估本地模型成熟度
- Agent 設計 model-agnostic(現在這樣寫 OK,LibreChat 支援)
- 不要設計成「Claude 專屬」

## 代理 29 · Claude(AI 本身的自省)

### 🟡 問題 M-1:「高迎合性」風險
**自省**:我作為 10 位承富同仁每天使用的 AI:
- 若我被設定成「永遠客氣、永遠說『好喔這是很棒的想法』」
- 承富同仁會逐漸依賴我的肯定 → 降低批判性思考
- 特別是年輕同仁,可能把 AI 意見等同權威

**修正**:
- Agent prompt 強化「**誠實回饋 > 討好**」:
  - 「若使用者的方向明顯有問題,直接指出」
  - 「不用『很棒的想法』這類空讚美」
- Sterio 在訓練時強調「AI 是協作者不是 yes-man,它該挑戰你」

### 🟡 問題 M-2:「Sycophancy」vs 「承富的權威層級」
**微妙衝突**:
- AI 應對所有使用者「同等誠實」
- 但承富有層級(老闆 vs 新人)
- 新人可能希望 AI 替他「擋」老闆的無理要求
- 老闆可能希望 AI 「教育」新人

**修正**:這不是 AI 能解的,是人的問題。但 Sterio 要意識到:AI 不該被武器化成職場工具。

### 🟡 問題 M-3:AI 作為「第 11 位同事」的比喻可能誤導
**反思**:本案 tagline「承富的第 11 位同事」
- 好處:好理解、好行銷
- 壞處:同事會受訓練、會成長、會離職——AI 不會
- 若承富老闆真把 AI 當成同事,可能:
  - 期待它理解情境(它不會)
  - 把它當責任歸屬(法律上不能)
  - 忽視持續調校(會壞但不會哀號)

**修正**:
- 訓練時強化:「AI 是工具,不是同事。我們用這個比喻是溝通方便,不是實質關係」
- 簡報結尾加 disclaimer:「比喻為促進理解,實際為軟體系統」

## 代理 30 · 未來 6 個月(事後諸葛)

### 🔴 預言 Z-1:什麼會第一個壞掉?
**最有可能**(機率降序):
1. **第 3 週 Day 17:某個 Agent prompt 寫錯,大量產出糟糕建議書**(Q-1 問題)
2. **第 4 週開訓練那天:網路不穩,線上演示失敗**(N-2 問題)
3. **第 6 週:某同仁上傳整個案件資料夾(50 檔)一次處理,爆 rate limit**(A-1 問題)
4. **第 8 週:承富老闆發現 AI 產出的新聞稿有常用錯誤,但不知道怎麼調**(Q-3 問題)
5. **第 12 週:Mac mini SSD 空間滿,LibreChat 當機**(備份策略缺失)

**修正**:針對每個預言提前準備應對 playbook:
- Agent 出錯 → 快速 rollback 到上一版 system prompt
- 網路不穩 → 訓練備用離線投影材料
- Rate limit → pre-warning dashboard
- 無法調 AI → Champion 寫簡易 prompt tuning 教學
- SSD 滿 → 自動老檔歸檔腳本

### 🟡 預言 Z-2:哪種同仁會「偷偷不用」?
**最可能不用的三類**:
- **45+ 資深同仁**:覺得自己不需要、不好意思問
- **極度 perfectionist**:覺得 AI 寫的都不夠好,自己重寫更快
- **被 AI 批評過的人**:負面體驗後迴避

**修正**:
- 首月每週用量報告,找出「低用量」同仁
- 主動 1:1 理解原因(不是要求使用,是傾聽)
- 調整 Agent prompt 或提供個人化協助

## 代理 31 · 倫理 / 社會影響

### 🟡 問題 ET-1:「省下 280 hr」的時間流向
**倫理問題**:AI 替承富省 280 hr/月。這時間流向哪?

| 情境 | 意義 |
|---|---|
| A · 同仁下班更早 | 生活品質提升,同仁支持 AI |
| B · 做更多案子 | 公司營收增、獎金增、同仁也 OK |
| C · 老闆裁員 2 人 | 同仁抗拒 AI、採納率歸零、系統廢掉 |
| D · 給更少薪(因工時短) | 同仁離職潮、承富聲譽毀損 |

**實情**:承富若走 C 或 D,這套系統會**被內部破壞**(消極抵制、輸入廢資料、故意不用)。

**修正**:
- Sterio 與老闆明確對話:「這套系統的意義是讓承富做更多、更好,不是裁減人力」
- 老闆在 Week 0 溝通會的公開承諾(C-2 問題有提)
- 首 12 個月承諾不縮編

### 🟡 問題 ET-2:AI 寫的內容署名誰?
**實情**:承富新聞稿送出去,署名「承富創意整合行銷」。背後 70% 是 AI 寫的,30% 是編輯的。
- 讀者以為是「人」寫的
- 這算欺瞞嗎?

**修正**:
- 參考國際 AI 內容揭露趨勢(尚未強制)
- 建議在新聞稿**底部**加一行小字:「本文採用 AI 協助撰寫,由承富編輯審訂」
- 這不是必要,但是品牌誠信姿態

## 代理 32 · 魔鬼代言人(Devil's Advocate)

### 🔴 問題 DA-1:這東西真的會被用嗎?
**拷問**:過去 10 年,多少企業內部建置的「數位工具」真正被用?
- 統計:大型軟體導入專案失敗率 30-50%
- 最常見失敗原因:**工具設計完美,但沒人用**

**拷問承富**:
- 10 人公司,半年後真實每日使用者會是幾人?最可能是 3-5 人
- 資深同仁可能從第一天就不碰
- 小編可能只用一個 Agent(社群貼文)
- 老闆可能用兩週就忘

**修正**:在 ROI 預估用**極保守情境**:假設 5 人 / 月用 30 小時 = 150 hr/月節省,折 NT$ 7 萬/月。**即使這樣,年度仍回本。** 這才是可信承諾。

### 🔴 問題 DA-2:Sterio 的單點故障 + 利益衝突
**拷問**:
- Sterio 是這整套系統**唯一懂的人**
- 同時收 NT$ 88K 顧問費 + 每月 NT$ 5K 維運費
- 他會希望這系統複雜(保飯碗)還是簡單(讓承富自立)?
- 若承富想自己接手,Sterio 配合度多高?

**修正**:
- 合約明文「Sterio 不得刻意設計複雜讓承富無法自主」
- Week 4 交付時,承富 AI Champion 要能獨立做 80% 維運
- Sterio 的價值應該體現在「優化升級、策略建議」,不是「救火基本問題」

### 🟡 問題 DA-3:4 個月後承富停止維運會怎樣?
**情境推演**:Month 4 承富決定停維運 NT$ 5K/月。
- Mac mini 還在跑 ✓
- Agent 還能用 ✓
- 6 個月後 LibreChat 有重大安全更新,沒人推 → 有漏洞 ✗
- 8 個月後 Claude 4.x 模型被 deprecated,Agent 開始報錯 ✗
- 12 個月後 Anthropic 改 API 格式,完全掛 ✗

**修正**:
- 合約附件「無維運的風險」清單
- 建議承富至少保「最低維運」NT$ 1,500/月(只處理緊急故障)
- 如果完全斷,12 個月後系統必然死

## 代理 33 · Sterio 的未來自己(12 個月後的回顧)

### 🟡 問題 SF-1:「我當初該但沒做的事」
**時光回望(虛擬)**:
1. 沒在 Week 0 建一份「**承富的品牌 voice 定義書**」,結果每個 Agent 各自摸索,3 個月才收斂風格
2. 沒把知識庫灌入列為「**承富自己的責任**」,結果變成 Sterio 每月要幫忙整理新檔案,維運工時爆
3. 沒有簽「**Exclusive clause**」,結果承富 6 個月後把案例學走,自己找人做第二家同業
4. 沒在 Agent prompt 放「**否決線**」(例:碰到 XX 情境必退回人工),結果 AI 處理了根本不該處理的事
5. 沒建「**月度使用報告 auto-email 給老闆**」,老闆感覺不到 AI 價值,半年後不續約

**修正**:
- Week 0 定義書(Q-3 問題)
- 承富 onboarding 自己灌 KB(簡易 SOP + UI 工具)
- 合約討論 exclusivity 條款(有彈性)
- Agent prompt 加「否決線」
- 自動化月度報告寄送

### 🟡 問題 SF-2:這個案子對 Sterio 個人的意義
**反思**:
- 若成功:Sterio 建立「10 人規模 AI 系統導入」的口碑,後續接案順暢
- 若失敗:可能失去一個重要早期客戶,口碑受損
- 中間:承富半滿意,一年後不續約但互道客氣

**修正**:
- 不是技術議題,但影響 Sterio 如何分配精力
- 建議 Sterio 把承富案**寫成 case study**(匿名化)
- 未來接下一案時帶著承富的 lessons learned
