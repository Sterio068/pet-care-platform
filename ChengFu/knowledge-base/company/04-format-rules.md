# 承富格式規則(金額 / 日期 / 數字 / 單位)

AI 產出**所有**對外文字必遵守這些格式。不一致 = 不專業。

## 一、金額

### 1.1 新台幣

| 場景 | 格式 | 範例 |
|---|---|---|
| 一般 | `NT$ 12,345` | NT$ 45,678 |
| 超過百萬 | `NT$ 1,234,567`(仍用逗號,不用「萬」「百萬」混) | NT$ 3,000,000 |
| 簡寫(社群 / 內部) | `NT$ 300 萬` | 但**提案/合約用精確數字** |
| 預算級距(Go/No-Go 初評) | `NT$ 數百萬等級` / `NT$ 千萬等級` | |

**絕不用**:
- ❌ `3 millon NTD` `TWD 3,000,000`(用 NT$)
- ❌ `30 萬台幣`(「台幣」多餘)
- ❌ `300,0000`(位數錯)

### 1.2 外幣

| 幣別 | 格式 | 範例 |
|---|---|---|
| 美元 | `USD 1,234.56` | USD 50.00 |
| 人民幣 | `RMB 1,234` | RMB 3,000 |
| 日圓 | `JPY 10,000` | JPY 50,000 |

### 1.3 金額搭配時間

- 月租:`NT$ 5,000 / 月` 或 `月 NT$ 5,000`
- 年度:`NT$ 120,000 / 年`
- 時薪:`NT$ 500 / 小時`(承富內部試算用)

### 1.4 含稅 vs 未稅

- 預設 **未稅**(政府標案常態)
- 若為含稅,**明確標註** `NT$ 12,345(含稅)` 或 `NT$ 12,345(未稅)`
- 對外報價務必清楚,避免爭議

## 二、日期

### 2.1 標準格式

| 場景 | 格式 | 範例 |
|---|---|---|
| 正式文書(政府) | 民國年月日 | 民國 115 年 4 月 18 日 |
| 一般(企業 / 媒體 / 對內) | 西元年月日 | 2026 年 4 月 18 日 |
| 簡寫(內部 / 社群) | YYYY/MM/DD | 2026/04/18 |
| ISO(資料欄位) | YYYY-MM-DD | 2026-04-18 |

### 2.2 規則

- 承富預設 **西元年**
- 若原文用民國年(政府來函)→ **保留民國年**,不擅自換算
- 若給政府客戶的正式文書 → **優先民國年**(除非客戶要求西元)
- 不同場景要**一致**:整份文件用西元就全用,不混用

### 2.3 期程表達

| 場景 | 寫法 |
|---|---|
| 確定日期 | 「2026 年 4 月 18 日(週六)」(週次可加括號) |
| 區間 | 「2026 年 4 月 18 日 ~ 2026 年 5 月 10 日」 或「~ 5/10」 |
| 大致 | 「2026 年 Q2」「2026 年 4 月下旬」 |
| 期程 | 「自開工日起 90 個日曆天」(標案常用) |
| 相對 | 「完工後 30 天內」「驗收通過後 15 個工作日內」 |

### 2.4 時間

- 24 小時制:`14:30`(不是 `下午 2:30`)
- 正式場合:`下午 2 時 30 分`
- 時區:台灣預設 CST,跨國專案標註 `(GMT+8)`

## 三、數字

### 3.1 阿拉伯數字 vs 中文

| 場景 | 用法 |
|---|---|
| 金額、數量、比例 | 阿拉伯數字(25%、1,234 人) |
| 序數(第 N 次)、名次 | 可中文(「第五屆」)或阿拉伯(「第 5 屆」),**一份文件一致** |
| 法規條文 | 中文(「第一條第二項」) |
| 章節編號 | 阿拉伯(「第 3 章 第 2 節」) |
| 文學化描述 | 中文(「承富十年磨一劍」) |

### 3.2 千分位與小數

- 千分位:`1,234,567`(每 3 位加逗號)
- 小數:**必要才用**(「3.5%」「2.8 小時」),多餘 0 不加(`5` 不是 `5.0`)

### 3.3 比例

| 寫法 | 用途 |
|---|---|
| `25%` | 一般 |
| `1:3` | 比例對照 |
| `每 4 人 1 位 PM` | 文字敘述 |
| `四分之一` | 文學化 |

## 四、單位

### 4.1 常用

| 類型 | 正確 | 錯誤 |
|---|---|---|
| 重量 | 公斤 / 公噸 / kg | kilo、千克(避免) |
| 長度 | 公尺 / 公里 / m / km | 米(避免) |
| 面積 | 平方公尺、坪 | 平米(避免) |
| 容量 | 公升 / 毫升 | 升、毫升(可) |
| 容量(資料) | GB / MB / TB | Gb(小 b 是 bit,錯) |

### 4.2 人員

- **人數**:「10 位同仁」「5 位工作人員」(不是「10 個人」)
- **人次**:活動常用(「500 人次參與」= 可能同一人多次)

## 五、標點

### 5.1 全形 vs 半形

- 中文內**全形**:,。!?;:「」『』()
- 中英混排:英文字母與數字兩側用半形,但緊接的標點**看語境**

### 5.2 破折號與省略號

- 破折號:`——`(兩個全形「—」)
- 省略號:`……`(兩個全形「…」)
- 不要用 `...` 或 `--`

### 5.3 引號

- 第一層:「」
- 第二層:『』(「XXX『YYY』ZZZ」)

## 六、列表與編號

### 6.1 正式文書

```
一、第一點
  (一)第一子點
    1. 第一項
      (1) 第一子項
        甲、…
```

### 6.2 一般文件

```
• 第一點
  ◦ 子點
    - 子子點
```

### 6.3 Markdown(內部 / 網頁)

用 `-` 或數字 `1.` 即可,不硬塞正式文書格式。

## 七、檔案命名(承富內部慣例)

```
YYYYMMDD-客戶簡稱-案名-版本-承辦人.ext
```

例:
- `20260418-環保局-AI宣導-v3-王PM.docx`
- `20260418-遠傳-活動策劃-final-陳PM.pptx`

## 八、AI 自我檢查流程

每次產出對外文字,逐項檢查:
- [ ] 金額格式:`NT$ XX,XXX`
- [ ] 日期格式:正式 → 民國;一般 → 西元
- [ ] 千分位逗號
- [ ] 單位正確且一致
- [ ] 中英標點符號不混
- [ ] 列表格式符合文件類型

發現不符 → 自行修正 → 才輸出。
