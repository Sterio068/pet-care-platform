# 政府標案結構分析

## 何時用

- 承辦拿到招標須知 PDF(可能 30-60 頁)
- 需要**10-15 分鐘**內讀出重點以便進行 Go/No-Go 評估
- 作為撰寫服務建議書的前置分析

## 承富的 9 欄閱讀法(必讀順序)

每一份政府招標 PDF 都用以下 9 欄結構化:

### 1. 承辦機關與聯絡人
- **機關全銜**(驗證單位,避免誤讀)
- **承辦單位**(具體科室,例:新聞傳播科)
- **聯絡人** + 電話 + Email
- **採購窗口**(可能與承辦不同人)

**承富經驗**:第一通電話打給承辦最重要。許多「模糊的條款」透過口頭澄清會變清楚。

### 2. 案名與案號
- **完整案名**(用於追蹤與決標公告檢索)
- **案號 / 標案編號**(政府電子採購網查用)
- **招標方式**:公開招標 / 公開取得 / 選擇性招標 / 限制性招標
- **決標方式**:最有利標 / 最低價 / 複數決標 / 統包

### 3. 預算金額
- **總額**(未稅 / 含稅)
- **細項分配**(人力 / 材料 / 設備 / 管理 / 其他)
- **履保金**:比例(通常 5-10%)與退還時點
- **期中 / 期末付款比例**(承富現金流關鍵)

**承富經驗**:政府案預算若低於 70% 市場行情,通常執行會非常辛苦。若低於 50% → 直接 No-Go。

### 4. 決標方式
- 評分方式(商業考量 / 技術考量的比例)
- 評審委員組成(機關內部 / 外部學者 / 業界)
- 簡報時機(書面審查 / 簡報答辯 / 兩階段)

### 5. 廠商資格
- **資本額**門檻
- **實績要求**(近 X 年內完成 X 個類似案)
- **證照 / 執照**(特定產業用)
- **ISO、CIS 等認證**
- **分包限制**(不可分包 vs 可分包比例)

**承富經驗**:資格查最快(10 分鐘內看得出能不能投)。若實績要求「公部門宣導類案 3 件以上、每件 100 萬以上」,承富 10 年累積足夠,但新公司就會卡關。

### 6. 評分項目與配分
以表格呈現:
| 項目 | 配分 | 承富弱點 | 承富強項 |
|---|---|---|---|
| 技術能力 | 30% | | 政府案經驗 |
| 執行規劃 | 25% | 時程緊時可能不夠具體 | |
| 團隊配置 | 15% | | |
| 價格 | 20% | 承富通常非最低 | |
| 其他 | 10% | | |

### 7. 工作交付物
- **主要工作項目**(活動場次、刊播量、報告份數等)
- **交付物類型**(PDF 報告 / 實體印刷 / 影音 / 網站等)
- **對應數量與時間**(哪個月交什麼)
- **驗收標準**(量化或模糊)

### 8. 期程罰則
- **開工日**(決標後 X 天內)
- **各階段里程碑**
- **驗收標準**(量化 vs 模糊)
- **延遲罰則**(通常每日總額千分之一到千分之三)
- **違約罰款**(重大違約的上限)

### 9. 潛在風險點(主動分析,非原文)
AI 需主動識別:
- 資格門檻過嚴(可能是指定廠商?)
- 預算與工作量不成比例
- 期程緊迫(週末工作風險)
- 評審委員偏好不明(看主辦機關過往決標對象)
- 履約爭議可能(工作項模糊、變更頻繁)

## 輸出格式

表格 + 條列,讓 PM 一眼看完決定方向:

```markdown
# 招標案重點表:XX 標案 2026

## 1. 基本資訊
- 機關:[全銜]
- 案號:[案號]
- 預算:NT$ [金額](未稅)
- 期程:[日期 ~ 日期]

## 2-8. [依上述結構]

## 9. 承富評估
- 資格符合:✅ / ⚠️ / ❌
- 建議方向:[Go / No-Go / 待釐清]
- 3 個關鍵風險點:
  1. ...
  2. ...
  3. ...

## 📌 下一步
- [ ] 打電話給承辦釐清 XX
- [ ] 調閱過往類似案的結案報告
- [ ] 執行 Go/No-Go 評估(Skill 02)
```

## 承富過往案例參考(few-shot)

**環保署 2024 海洋廢棄物宣導案**:
- 承富決策:**Go**
- 理由:預算合理(NT$ 380 萬)、資格符合(承富 3 個類似實績)、期程 6 個月可執行
- 結果:得標,毛利率 22%(符合承富目標)

**文化局 2023 社區藝文推廣案**:
- 承富決策:**No-Go**
- 理由:預算偏低(NT$ 220 萬要辦 12 場活動 + 記者會 6 場),實質單位成本不足
- 結果:未投,該案由另一家低價搶標得標,後續傳出違約糾紛,證明決策正確

## 自我檢查

輸出前確認:
- [ ] 9 欄都填(不能有空)
- [ ] 金額用 NT$ 格式
- [ ] 日期格式符合(民國年保留 / 西元年統一)
- [ ] 風險點至少 3 個(主動識別)
- [ ] 建議方向明確(不要說「可以考慮」這種模糊話)
