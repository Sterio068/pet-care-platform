> 這是系統整體的**設計語言、資訊架構、UX 流程**規範。
> 先讀這份,再看實作細節(CLAUDE.md、ARCHITECTURE.md、tasks/)。

## 1. 設計哲學(4 條核心原則)

取自 Apple Human Interface Guidelines 並對應本系統需求:

### 1.1 Clarity(清晰)
- 每個畫面、每個功能,使用者**3 秒內**知道這在做什麼
- 不用裝飾性元素掩蓋資訊,不用花俏動畫拖慢節奏
- 字型、顏色、間距建立**穩定的視覺階層**

### 1.2 Deference(退讓)
- 系統介面退居幕後,**內容(對話、產出)永遠主角**
- 側邊欄半透明、薄、可收折
- 按鈕素雅,不搶眼
- 空間留白勝過擠滿資訊

### 1.3 Depth(層次)
- **4 層視覺深度**:背景 → 內容層 → 浮動層 → 模態層
- 沒有層次 = 所有元素看起來一樣重要 = 使用者無法判斷焦點
- 用微陰影、毛玻璃、Z-index 建立層次感

### 1.4 Discoverability(可發現性)
- 重要功能**不藏**:首頁、側邊欄、Menu Bar(⌘K)三種路徑
- 使用者偶然點到也能找到
- 搜尋無所不在(⌘K 呼出)
- 鍵盤快速鍵可發現(第一次按 ⌘ 有 tooltip 提示)

## 2. 資訊架構(Information Architecture)

### 2.1 從「29 個 Preset」轉型成「5 個 Workspace」

**為什麼要 Workspace?**

不是把 29 個 preset 分 5 類放在選單上就叫「資訊架構」。
Workspace = 使用者**一整個任務情境**,進去就知道可以做什麼、通常下一步是什麼。

這符合人類心智模型(Mental Model):使用者不是來「找工具」,是來「完成任務」。

### 2.2 5 個 Workspace

| Workspace | 情境 | 面對對象 | 典型任務流 | 收錄 Preset |
|---|---|---|---|---|
| 🎯 **投標** | 看到招標須知到送件 | PM、業務、老闆 | 評估 → 解析 → 建議書 → 簡報 | 25, 01, 02, 24, 22 |
| 🎪 **活動執行** | 從規劃到驗收 | PM、設計、現場 | 3D 模擬 → 舞台 → 動線 → 現場體驗 → 廠商 → 委外合約 | 09, 11, 12, 13, 10, 14 |
| 🎨 **設計協作** | PM ↔ 設計師溝通 | 設計師、PM | 發想 → Brief → 圖像 → 適配 → 系統 | 20, 21, 27, 28, 29 |
| 📣 **公關溝通** | 對外發聲 | 小編、PM | 新聞稿、社群、Email、會議 | 04, 05, 23, 26, 06 |
| 📊 **營運後勤** | 內部管理 | PM、財務、HR | 結案 → 報價 → 追蹤 → CRM → 合約 → 稅務 → 新人 → 知識庫 | 03, 15, 16, 17, 18, 08, 19, 07 |

各 Workspace 設計成**完整閉環**:使用者進去,能依工作流程順序完成所有任務,不用跨 Workspace 切換。

### 2.3 Workspace 內部結構

每個 Workspace 有:
- **封面 Cover**:說明這個 Workspace 是做什麼的(1 行)
- **主要流程 Primary Flow**:標準 3-5 個 preset,按使用順序排列
- **進階工具 Advanced**:較少用但有時需要的 preset
- **常用模板 Templates**:客戶、機關、活動類別的既有模板
- **近期對話 Recent**:在這個 Workspace 下最近開過的對話

**範例:投標 Workspace**

```
╭─────────────────────────────────────────────╮
│  🎯 投標                                      │
│  從看到招標須知,到送件前的最後一哩             │
╰─────────────────────────────────────────────╯

▶ 從零開始 · 標準流程

  ① Go/No-Go 評估  ──→  ② 解析招標須知
  要不要接?              60 頁 PDF → 結構化

  ③ 建議書初稿   ──→   ④ 競品視覺研究
  Word 初稿 1 hr 完成    找差異化機會

  ⑤ 投標簡報架構
  15-25 頁結構 + 視覺指引

▶ 進階工具
  • 舞台/動線規劃(若活動型標案)
  • 客戶 CRM 查此機關過往互動

▶ 本 Workspace 近期對話
  • 2024 環保局招標解析(3 小時前)
  • 新竹市府採購案 Go/No-Go(昨天)
```

### 2.4 全域工具(不屬於任何 Workspace)

以下 Preset 跨 Workspace 都會用到,設為**全域工具**,從任何地方都能喚出:

- **公司知識庫查詢**(07)— 任何時候都可能需要查過往案
- **Email / 客戶溝通**(26)— 不屬於特定流程
- **會議速記**(06)— 會議不限 Workspace
- **稅務/合約諮詢**(08)— 突發需求
- **新進同仁 Onboarding**(19)— 新人專用

## 3. UX 流程(3 個關鍵旅程)

### 3.1 首次登入旅程

```
[1] 登入頁 — 輸入 Email + 密碼
     ↓
[2] 首次歡迎 — 半秒動畫,介紹「這是承富 AI 系統」
     ↓
[3] 角色選擇 — 「您是:PM / 設計 / 小編 / 財會 / 業務」
     (決定首頁預設 Workspace)
     ↓
[4] 首頁 Dashboard — 顯示該角色最常用 3 個 preset + 介紹 Workspace 概念
     ↓
[5] 建議:「試試看『招標解析』Demo」(30 秒快速體驗)
```

### 3.2 日常使用旅程

```
[1] 登入 → 首頁
     ↓
[2] 選其一:
    A. 點最近對話 → 接續之前工作
    B. 點 Workspace 卡片 → 進入情境
    C. ⌘K → 全域搜尋直接找 preset
     ↓
[3] Workspace 內 → 看到建議的下一步
     ↓
[4] 點 preset → 開新對話
     ↓
[5] 完成任務 → 回首頁或繼續下一步
```

### 3.3 跨 Workspace 任務

案例:PM 正在寫某客戶的「結案報告」(營運 Workspace),發現要引用過往「新聞稿」(公關 Workspace)。

```
情境:PM 在 結案報告 對話中,輸入「/」
     ↓
彈出 slash-command 選單:
     ↓
/knowledge(知識庫查詢)
/email(email 草稿)
/vendor(廠商比價)
     ↓
PM 選 /knowledge → inline 查詢面板
     ↓
結果自動貼回當前對話,不用切分頁
```

這叫 **inline tool switching**,避免使用者在 tab 間切換時失去脈絡。

## 4. 視覺語言(Visual Language)

### 4.1 色彩系統

**主色(Primary / Accent)**
- Navy 深藍 `#0F2340` — 系統品牌色,用於 CTA、標題、重要元素
- 使用者可在設定改為 macOS System Accent Color(藍/紫/粉/橘/黃/綠/石墨)

**語意色(Semantic)**
| 用途 | Light mode | Dark mode |
|---|---|---|
| 成功 | `#34C759` macOS Green | 同 |
| 警告 | `#FF9500` macOS Orange | 同 |
| 錯誤 | `#FF3B30` macOS Red | 同 |
| 資訊 | `#007AFF` macOS Blue | 同 |

**背景層次(Layered Backgrounds)**
| 層級 | Light | Dark |
|---|---|---|
| 最底層 | `#F5F5F7` | `#1C1C1E` |
| 內容層 | `#FFFFFF` | `#2C2C2E` |
| 浮動層 | `rgba(255,255,255,0.8)` + blur(20px) | `rgba(44,44,46,0.8)` + blur(20px) |
| 模態層 | `#FFFFFF` + shadow | `#3A3A3C` + shadow |

**文字色**
| 層級 | Light | Dark |
|---|---|---|
| 主要 | `#1D1D1F` | `#F5F5F7` |
| 次要 | `#6E6E73` | `#AEAEB2` |
| 弱化 | `#C7C7CC` | `#48484A` |

### 4.2 字型系統

**英文 / 數字**:SF Pro Text / Display(macOS 原生)
**中文**:PingFang TC(macOS 原生)
**Fallback**:`-apple-system, BlinkMacSystemFont, "Noto Sans TC", sans-serif`
**等寬**:SF Mono / JetBrains Mono(對話中的程式碼、token 數字)

**字級階層**
| 用途 | Size | Weight | Line Height |
|---|---|---|---|
| Large Title(首頁歡迎) | 34px | 700 | 41px |
| Title 1(Workspace 名) | 28px | 700 | 34px |
| Title 2(段落標題) | 22px | 600 | 28px |
| Title 3(卡片標題) | 20px | 600 | 25px |
| Headline | 17px | 600 | 22px |
| Body(內文) | 15px | 400 | 21px |
| Callout(說明) | 14px | 400 | 19px |
| Subhead(副標) | 13px | 400 | 18px |
| Footnote | 12px | 400 | 16px |
| Caption | 11px | 400 | 13px |

### 4.3 間距系統(8pt Grid)

所有間距 = 4px 的倍數。標準值:
- `4px` — 極小(icon 與文字間)
- `8px` — 小(按鈕 padding)
- `12px` — 中(卡片內距)
- `16px` — 常用(元素間距)
- `24px` — 段落間距
- `32px` — 區塊間距
- `48px` — 主要段落切分
- `64px` — 大段落

### 4.4 圓角系統

- `4px` — 標籤、小按鈕
- `6px` — 輸入框、下拉選單
- `8px` — 按鈕、卡片
- `10px` — 模態、浮動面板
- `12px` — 大卡片(Workspace 卡)
- `16px` — 最大容器

### 4.5 陰影系統(層次感)

**Light mode**
```css
/* 內容層陰影 */
--shadow-1: 0 1px 3px rgba(0,0,0,0.04), 0 1px 2px rgba(0,0,0,0.06);
/* 浮動層陰影 */
--shadow-2: 0 4px 12px rgba(0,0,0,0.08), 0 2px 4px rgba(0,0,0,0.04);
/* 模態層陰影 */
--shadow-3: 0 20px 48px rgba(0,0,0,0.15), 0 8px 16px rgba(0,0,0,0.06);
```

**Dark mode** 使用更深、更大範圍的陰影。

### 4.6 圖示系統

**規範**:一律使用 **SF Symbols 風格** — 線條圖示,粗細一致(1.5 stroke),圓弧銜接。

**來源**:
- 若可載入 SF Symbols(僅 macOS/iOS Safari):直接用
- 否則:使用 Phosphor Icons(開源、風格最接近 SF Symbols)
- **禁止**:Font Awesome Solid 風格、Material Icons、擬物化圖示

**Workspace 圖示**(SF Symbols 名稱 / Phosphor 替代)
- 投標 → `target` / Phosphor Target
- 活動執行 → `calendar.badge.plus` / CalendarPlus
- 設計協作 → `paintpalette` / Palette
- 公關溝通 → `megaphone` / Megaphone
- 營運後勤 → `chart.bar.doc.horizontal` / ChartBar

### 4.7 動畫原則

**原則:快、輕、有意義**

- 標準時長:`200ms`(不超過 `300ms`)
- Easing:`cubic-bezier(0.4, 0, 0.2, 1)`(macOS-like)
- 進入動畫:fade + 輕微位移(8-16px)
- 退出動畫:fade-out(不加位移)
- **禁止**:彈跳、旋轉、縮放過大、顏色漸變

## 5. 互動模式(Interaction Patterns)

### 5.1 鍵盤快速鍵

**全域**
- `⌘K` 呼出指令面板(Command Palette)
- `⌘N` 新對話
- `⌘,` 系統設定
- `⌘/` 顯示鍵盤快速鍵表
- `⌘Shift+L` 切換深色 / 淺色模式
- `esc` 關閉浮動面板 / 退出全螢幕

**Workspace 切換**
- `⌘1` 投標
- `⌘2` 活動執行
- `⌘3` 設計協作
- `⌘4` 公關溝通
- `⌘5` 營運後勤
- `⌘0` 回首頁

**對話內**
- `⌘Enter` 送出訊息
- `⌘⇧N` 開新對話(保留當前)
- `⌘F` 對話內搜尋
- `⌘⇧F` 全域搜尋所有對話
- `⌘↑ / ⌘↓` 切換對話歷史

**文件 / Preset 使用**
- `⌘P` 打印 / 匯出當前對話
- `⌘E` 匯出為 Word
- `⌘S` 收藏對話

### 5.2 指令面板(Command Palette)

**觸發**:`⌘K`

**內容**:
```
┌──────────────────────────────────────────┐

├──────────────────────────────────────────┤

│  🎨 主視覺概念發想        設計 > 發想       │
│                                           │
│  近期對話                                  │
│  2024 環保局招標(3 小時前)                 │
│  客戶 A 月度社群企劃(昨天)                 │
│                                           │
│  知識庫                                    │
│  「公司組織圖」找到 3 筆                    │

```

**設計原則**:
- Fuzzy search(模糊搜尋)— 支援拼音、縮寫
- **結果分組**:功能 / 對話 / 知識庫 / 動作
- 方向鍵導航,Enter 執行
- 顯示快速鍵提示(如 `⌘K` `↑↓ 移動` `Enter 執行`)

### 5.3 Slash Commands(對話內)

**觸發**:在對話中輸入 `/`

**用途**:不離開當前對話情境下快速呼叫其他 preset

**範例**:
```
使用者正在「結案報告」對話中
輸入 "/know 環保局去年"
     ↓
自動呼叫「知識庫查詢」preset 做 inline 查詢
     ↓
結果以引用區塊貼回當前對話
     ↓
使用者繼續寫結案報告,脈絡不斷
```

**支援的 slash 命令**:
- `/know [查詢]` 知識庫查詢
- `/email [情境]` email 草稿
- `/vendor [類別]` 廠商比價
- `/tax [問題]` 稅務諮詢
- `/help` 顯示可用命令

### 5.4 右鍵選單 / 二次操作

對話內的文字訊息,右鍵可:
- 複製(⌘C)
- 收藏到知識庫
- 匯出成 Word 段落
- 讓 AI 重新生成(不滿意)
- 翻譯成英文(給客戶看)

### 5.5 模態 Sheet(聚焦任務)

複雜的輸入(如「建議書助手」需要大量背景資訊)用**Sheet 模態**(macOS 風格從頂部滑下),而不是打開新分頁。

Sheet 特徵:
- 從頂部 8% 處向下滑出
- 覆蓋原畫面 70-80% 高度(留邊露底)
- 背景暗化 + 毛玻璃
- 無邊框、無標題列(macOS Big Sur 以後風格)
- `esc` 或點擊背景關閉

## 6. 漸進式揭露(Progressive Disclosure)

**原則**:使用者「第一次看到」vs「熟練使用」vs「進階操作」看到的東西不同。

### 6.1 首頁漸進式

| 使用天數 | 使用者看到 |
|---|---|
| Day 1 | 大標「歡迎 [名字]」+ 5 個 Workspace + 新手教學提示 |
| Day 2-7 | 「最近對話」卡 + 「本週常用」出現(還不足就顯示 Workspace 介紹) |
| Day 8+ | 「常用 3 個 preset」放最上面 + 「智慧建議」依時段出現(例如週一 9:00 推「本週待辦摘要」) |

### 6.2 Preset 本體漸進式

每個 Preset 的 system prompt 是完整且嚴謹的(已設計好)。
但「使用者需要輸入什麼」可以漸進揭露:

**入門模式**:顯示 2-3 個必填欄位
**進階模式**:展開 5-10 個選填欄位(Temperature、模型、輸出格式等)
**專家模式**:直接在輸入框改 system prompt

預設入門,使用者可在設定切到進階。

### 6.3 結果呈現漸進式

AI 回應的長內容,預設收折。
- 摘要 3 行顯示在上方
- 點「展開完整內容」才顯示全文
- 減少滾動、讓畫面保持輕盈

## 8. 錯誤處理 & 恢復

### 8.1 AI 回覆不滿意

對話訊息右下角出現:
- `👍` 讚
- `👎` 不好
- `🔄` 重新產生
- `✏️` 編輯後重試(改輸入內容)

### 8.2 API 暫時失敗

**絕不**顯示「Internal Server Error」
**要顯示**:
```
⚠️ 暫時無法連接 AI 服務
可能原因:網路不穩、Anthropic 服務忙碌

建議:
• 稍等 30 秒再試
• 改用 Haiku 模型(速度更快)
• 若持續發生,聯絡 AI Champion [姓名]
```

### 8.3 Token 超額警示

使用率 > 80%:
```
💡 您本月已用 80% token 配額(NT$ 6,400 / NT$ 8,000)
建議:大量文字生成改用 Haiku 模型,可省 80% 費用
[了解更多] [切換至 Haiku]
```

使用率 > 100%:
```
📢 本月 token 配額已用完
系統已自動切換到 Haiku 模型讓您繼續工作
需要更多配額?聯絡 AI Champion
```

## 9. 無障礙(Accessibility)

### 9.1 必備無障礙支援

- [ ] 所有互動元素可用 Tab 鍵到達
- [ ] 焦點明顯(2px 藍色 outline)

## 13. 承富系統設計的獨特性

為什麼這不是「把 LibreChat 套 macOS 主題就好」?

**我們做的是**:
1. **任務導向 IA**(5 Workspace)— 而非功能導向(29 個 preset 平鋪)
2. **漸進揭露**(新人 vs 熟手看到的不一樣)— 而非固定 UI
3. **上下文保留**(Slash commands inline 呼叫)— 而非分頁切換
4. **智慧建議**(基於使用模式推薦)— 而非手動找功能
5. **macOS-native 氣質**(毛玻璃、SF Symbols、鍵盤優先)— 而非 web-generic UI

結果:29 個功能變得「多而不雜」,使用者在任何時刻只看到與當前任務相關的部分。
