### 任務 3.1:匯入 Preset
```bash
cd ~/chengfu-ai
# LibreChat 提供 import API 或從 admin panel 匯入
for f in config-templates/presets/*.json; do
  # 使用 LibreChat API 或手動 admin panel 匯入
  echo "Importing $f"
done
```
- [ ] 29 個 Preset 在 admin panel 中可見
- [ ] 設定為「全公司可用」(不是特定用戶)

## Day 17-18:公司知識庫灌入(RAG)

### 任務 3.3:啟用 RAG API 容器
- [ ] 編輯 `config/docker-compose.yml`,取消 rag-api 服務的註解
- [ ] 設定 VectorDB:預設用 Chroma(輕量)
- [ ] `docker compose up -d rag-api vectordb`
- [ ] 確認:`curl http://localhost:8000/health` 應回 OK

### 任務 3.4:準備承富知識庫內容
從承富取得以下資料,放在 `knowledge-base/` 資料夾:

**必備(去識別化後)**:
- [ ] 過去 3 年服務建議書(優質版本,至少 5 份)
- [ ] 過去 3 年結案報告(代表作,至少 5 份)
- [ ] 公司品牌指南(視覺、口吻、禁用語)
- [ ] 標準合約範本
- [ ] 過往新聞稿優質範例(10-20 篇)

**選配**:
- [ ] 同業競品分析
- [ ] 產業趨勢報告
- [ ] 政府採購法相關文件
- [ ] 公司員工手冊、SOP

### 任務 3.5:檔案去識別化處理
對每份文件掃描並**替換以下內容**再灌入:
- [ ] 真實客戶姓名 → `[客戶 A]`、`[客戶 B]`
- [ ] 真實金額(若敏感) → `[NT$ 數萬等級]`
- [ ] 個資(電話、email、身份證) → `[已遮蔽]`

> 重要:這些檔案會送 OpenAI 做 embedding,雖然 OpenAI 承諾不訓練 API 資料,但謹慎為上。

### 任務 3.6:灌入 Vector DB
```bash
cd ~/chengfu-ai
python3 scripts/import-knowledge.py \
  --folder knowledge-base/ \
  --collection chengfu-kb \
  --chunk-size 500 \
  --overlap 50
```
- [ ] 確認處理日誌無錯誤
- [ ] 向量總數與預期符合(每份文件約 20-50 chunks)

### 任務 3.7:設定「公司知識庫查詢」Preset
- [ ] 打開 Preset 07,設定它預設連接 `chengfu-kb` collection
- [ ] 測試:問「我們去年環保局案的預算結構大概怎麼配?」
  - 應回傳基於承富過往結案報告的具體內容
- [ ] 測試:問「新聞稿的標題風格是?」
  - 應基於承富過往新聞稿給出分析
